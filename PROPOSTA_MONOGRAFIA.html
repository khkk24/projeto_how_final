1. TEMA

1.1 Delimitação do Tema

Desenvolvimento de um sistema inteligente de análise e predição de gravidade de acidentes de trânsito nas rodovias federais brasileiras, utilizando técnicas de ciência de dados e algoritmos de machine learning, baseado em dados históricos da Polícia Rodoviária Federal (PRF) do período de 2021 a 2025.

1.2 Justificativa da Escolha

A segurança no trânsito é um problema crítico de saúde pública no Brasil. Segundo dados da PRF, milhares de vidas são perdidas anualmente em acidentes nas rodovias federais. Este projeto propõe uma abordagem inovadora que combina:

  • Análise Exploratória de Dados: Para identificar padrões, tendências e fatores de risco
  • Machine Learning: Para prever a gravidade dos acidentes antes da chegada do socorro
  • Aplicação Prática: Com potencial de impacto direto na alocação de recursos de emergência

A relevância deste tema se justifica por:

  1. Impacto Social: Contribui diretamente para salvar vidas através de resposta mais eficiente a emergências
  2. Inovação Tecnológica: Aplica técnicas modernas de IA a um problema real e urgente
  3. Viabilidade: Utiliza dados públicos disponíveis e tecnologias consolidadas
  4. Originalidade: Combina análise descritiva com capacidade preditiva em um único sistema integrado

2. PROBLEMA DE PESQUISA

2.1 Contextualização

Os acidentes de trânsito nas rodovias federais brasileiras representam um grave problema de saúde pública. Entre 2021 e 2025, foram registrados 311.029 acidentes, resultando em 26.039 mortes e 355.066 feridos. A eficiência no atendimento às vítimas é crucial para reduzir a mortalidade, e para isso, é fundamental prever a gravidade do acidente no momento do acionamento.

2.2 Questão Problema Principal

“Como desenvolver um modelo preditivo capaz de classificar a gravidade de acidentes de trânsito em rodovias federais com base em características do acidente, condições ambientais e dados geográficos, visando otimizar a alocação de recursos de emergência?”

2.3 Questões Secundárias

  1. Quais características (features) são mais relevantes para prever a gravidade de um acidente?
  2. Qual algoritmo de machine learning apresenta melhor desempenho para este problema de classificação multiclasse?
  3. Como lidar com o desbalanceamento de classes (acidentes fatais representam apenas 7% dos casos)?
  4. Quais padrões temporais e geográficos podem ser extraídos dos dados históricos?
  5. Como traduzir as predições do modelo em ações práticas para equipes de resgate?

2.4 Hipóteses

H1: É possível prever a gravidade de acidentes de trânsito com acurácia superior a 75% utilizando dados disponíveis no momento do registro do acidente.

H2: Variáveis como tipo de acidente, localização (quilômetro), causa do acidente e horário são os principais preditores de gravidade.

H3: Modelos ensemble (Random Forest, Gradient Boosting) apresentam melhor desempenho que modelos lineares para este problema.

H4: Técnicas de balanceamento de classes (SMOTE, class weights) melhoram significativamente o recall para acidentes fatais.

Desenvolver e avaliar um sistema de análise e predição de gravidade de acidentes de trânsito nas rodovias federais brasileiras, utilizando técnicas de ciência de dados e machine learning, capaz de auxiliar na tomada de decisão para alocação eficiente de recursos de emergência.

5.2 Conceitos Fundamentais

5.2.1 Classificação Multiclasse

Problema de aprendizado supervisionado onde cada instância deve ser atribuída a uma dentre três ou mais classes. Neste projeto: - Classe 1: Sem Vítimas - Classe 2: Com Vítimas Feridas - Classe 3: Com Vítimas Fatais

5.2.2 Desbalanceamento de Classes

Situação onde as classes não têm distribuição uniforme. No dataset do projeto: - Com Vítimas Feridas: 76% - Sem Vítimas: 17% - Com Vítimas Fatais: 7%

Técnicas para lidar: - SMOTE (Synthetic Minority Over-sampling Technique) - Class Weights - Undersampling da classe majoritária - Métricas apropriadas (F1-Score, Recall)

5.2.3 Random Forest

Algoritmo ensemble que constrói múltiplas árvores de decisão e combina suas predições: - Reduz overfitting através de bagging - Fornece importância de features - Robusto a outliers - Paralelizável

5.2.4 Gradient Boosting

Algoritmo ensemble que constrói modelos sequencialmente, cada um corrigindo erros do anterior: - Alta performance em dados tabulares - Ajusta learning rate para convergência - Requer tuning cuidadoso de hiperparâmetros - Mais lento que Random Forest

5.3 Estado da Arte

5.3.1 Trabalhos Relacionados

Estudos prévios sobre predição de gravidade de acidentes:

  1. Abellán et al. (2013): Usaram árvores de decisão para prever gravidade de acidentes na Espanha, alcançando 71% de acurácia.

  2. Kwon et al. (2015): Aplicaram SVM e Random Forest em dados de acidentes na Coreia, com melhor resultado de 78% usando Random Forest.

  3. Santos et al. (2019): Analisaram dados brasileiros usando regressão logística, identificando hora do dia e tipo de via como principais fatores.

  4. Chen & Fan (2019): Utilizaram Deep Learning (CNN) para prever gravidade com 82% de acurácia, mas com dataset muito maior.

5.3.2 Lacunas Identificadas

  • Poucos estudos focados especificamente em rodovias federais brasileiras
  • Falta de sistemas integrados (análise + predição + interface)
  • Necessidade de abordagens práticas para implantação

5.3.3 Contribuição Deste Trabalho

6.3 Métricas de Avaliação

Acurácia: Proporção de predições corretas $$Accuracy = \frac{TP + TN}{TP + TN + FP + FN}$$

Precision: Proporção de predições positivas corretas $$Precision = \frac{TP}{TP + FP}$$

Recall (Sensibilidade): Proporção de positivos reais identificados $$Recall = \frac{TP}{TP + FN}$$

F1-Score: Média harmônica de precision e recall $$F1 = 2 \times \frac{Precision \times Recall}{Precision + Recall}$$

Matriz de Confusão: Visualização de acertos e erros por classe

6.4 Critérios de Sucesso

O projeto será considerado bem-sucedido se:

  1. Performance: Acurácia ≥ 75% e F1-Score ≥ 70%

  3. Reprodutibilidade: Pipeline completo documentado e executável
  4. Interface: Aplicação web funcional e intuitiva

Impacto: Alto
